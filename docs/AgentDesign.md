# 明确研究问题的重要性

> 在一个限定领域和显式知识结构中，Agent 能否通过少量自然语言交互，重建用户在各知识点上的掌握状态。

- 教学 Agent 需要知道用户哪里会、哪里不会；
- 解释型 Agent 需要决定解释深度；
- 编程助手需要判断用户能否理解某种抽象；
- 协作 Agent 需要知道哪些背景可以省略；
- 推荐和决策辅助系统需要理解用户已有知识。


# 实验验证的观点

## 子问题一：交互适应性

Agent 是否会根据交互中逐步获得的用户信息，改变后续诊断行为？

这里通过设置题库，设计三个实验

首先拆分 agent 模块，一个模块用来进行下一次提问决策，决定下一次问什么问题，另一个模块则用来根据回答结果进行 reconstructed map 的更新

三个实验分别是，从题库中随机抽取问题进行提问，然后更新；让 Agent 从题库中挑选问题提问，然后更新；让 Agent 直接提问，然后更新。三个实验的更新模块用相同的，仅改变提问模块

## 子问题二：用户模型驱动的适应

相比仅根据当前对话即时反应，维护一个持续演化的用户模型，是否能让 Agent 做出更连贯、有效的诊断决策？

一个持续用户模型能否帮助 Agent 累积证据、修正旧判断、保持跨轮一致性，并做出更好的后续决策。

这里设计两个实验，来测试用户模型的显示构造是否有助于 Agent 的提问决策选择。

一种采用每轮对话更新 reconstructed map，实时维护的方式来进行提问；另一种采用先进行对话提问，最后统一更新的方式进行

## 子问题三：贝叶斯角度优化提问策略

Agent 能否评估一次交互对最终用户状态重建的决策价值，并优先进行真正有助于最终判断的交互？

$$
R(h_t)
=
\min_{\hat Z}
\mathbb{E}
\left[
L(\hat Z,Z)\mid h_t
\right]
$$

其中：
- $Z$ 是隐藏 Knowledge Map；
- $\hat{Z}$是 Agent 最终提交的重建；
- $L$是当前 benchmark 的全图 mastery distance。

对于候选问题 $q$，定义它的价值：

$$KG(q\mid h_t)
=
R(h_t)
-
\mathbb{E}_{a\sim p(a\mid q,h_t)}
\left[
R(h_t,q,a)
\right]$$

也就是：
问完这个问题以后，预计最终全图重建误差能降低多少？

最后选择：
$$q_t^*=\arg\max_q KG(q\mid h_t)$$
